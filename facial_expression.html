Facial Expression classification using Electromyography (EMG)¶

Preview into dataset¶

EMG signals have been collected from recordings of 15 different subjects. The dataset is divided into training and test sets, where 12 subjects are used for training and 3 subjects for testing. Each recording (i.e., file) corresponds to a single facial expression and has a duration of 5 seconds.Recordings are categorized into three expression classes: “frown”, “pucker”, and “smile”. Each class contains multiple EMG recordings representing the corresponding facial muscle activity. The exact number of recordings per class may vary, but all three classes are present in both training and test subsets. Each EMG recording is stored as a CSV file and contains 4 columns, corresponding to 4 distinct EMG channels. These channels represent signals measured from different facial muscles: corrugator supercilii, zygomaticus major, orbicularis oris, and orbicularis oculi. The signals are sampled at 2048 Hz. The dataset is organized into separate “Train” and “Test” directories. Within each directory, there are three subfolders named “frown”, “pucker”, and “smile”, each containing the respective EMG recordings. The filename of each record includes both the event number and the subject ID. For example, in the filename “sample_ID01_0.csv”, “01” refers to the subject ID and “0” refers to the event number. Each subject contributes recordings across all three expression classes.

def load_dataset(base_path):
    """
    Load the data from the folder and save segment/label to train/test
    """
    X, y = [], []
    
    classes = ['frown', 'pucker', 'smile']
    
    for label, cls in enumerate(classes):
        files = glob.glob(os.path.join(base_path, cls, "*.csv"))
        
        for file in files:
            df = pd.read_csv(file)
            X.append(df)
            y.append(label)
    
    return X, y

X_train_raw, y_train = load_dataset("Data 5/train")

Processing¶

Preprocessing is done by using:

  • Notch filter at 50Hz to remove power line interference
  • Bandbass filter with fourth order to isolate physiologically relevant EMG frequency range. The 20 Hz low cutoff removes slow-moving artifacts such as electrode drift, motion artifacts, and cardiac interference, while the 450 Hz high cutoff discards high-frequency noise beyond the expected range of surface EMG
In [30]:
from scipy.signal import butter, filtfilt, iirnotch
import numpy as np

def notch_filter(signal, fs=2048, freq=50):
    b, a = iirnotch(freq, 30, fs)
    return filtfilt(b, a, signal)

def bandpass_filter(signal, fs=2048, lowcut=20, highcut=450, order=4):
    nyq = fs / 2
    b, a = butter(order, [lowcut/nyq, highcut/nyq], btype='band')
    return filtfilt(b, a, signal, axis=0)

def preprocess_df(df):
    processed = df.copy()
    
    for col in df.columns:
        processed[col] = notch_filter(df[col].values)
        processed[col] = bandpass_filter(df[col].values)
    
    return processed

X_train = [preprocess_df(df) for df in X_train_raw]
X_test  = [preprocess_df(df) for df in X_test_raw]

Feature extraction¶

  • time-domain features
    • mean
    • standard deviation
    • peak absolute amplitude
    • root mean square (RMS), especially important since it reflects the overall muscle activation level
    • waveform length
    • turns count, to measure the number of direction reversals in the signal above a small threshold, providing an indicator of signal irregularity associated with motor unit firing patterns
  • frequency-domain features (Using Welch's power spectral density estimate, which reduces noise variance compared to a simple FFT by averaging across overlapping signal segments)
    • Band power was extracted in four sub-bands (20–50 Hz, 50–100 Hz, 100–200 Hz, 200–400 Hz) to capture how muscle activation energy is distributed across the EMG frequency spectrum
  • wavelet energy
    • discrete wavelet composition (db4, 3 levels). The energy of each wavelet coefficient level was computed, capturing signal characteristics that vary across both time and frequency simultaneously. db4 wavelet is commonly used in EMG analysis due to its morphological similarity to EMG signal shapes.

from scipy.signal import welch
import pywt

def time_features(x):
    return [
        np.mean(x),
        np.std(x),
        np.max(np.abs(x)),
        np.sqrt(np.mean(x**2)),  
    ]

def waveform_length(x):
    return np.sum(np.abs(np.diff(x)))


def turns_count(x, threshold=0.01):
    diff = np.diff(x)
    turns = np.where((diff[:-1] * diff[1:] < 0) & (np.abs(diff[:-1]) > threshold))[0]
    return len(turns)


def bandpower(x, fs, fmin, fmax):
    freqs, psd = welch(x, fs=fs)
    idx = (freqs >= fmin) & (freqs <= fmax)
    return np.trapz(psd[idx], freqs[idx])


def wavelet_energy(x):
    coeffs = pywt.wavedec(x, 'db4', level=3)
    return [np.sum(c**2) for c in coeffs]


def extract_features(df, fs=2048):
    """
    Extracts the chosen features from the dataframe

    Returns features as a numpy array
    """
    features = []
    
    for col in df.columns:
        x = df[col].values
        
        # Time-domain features
        features.extend(time_features(x))
        features.append(waveform_length(x))
        features.append(turns_count(x))
        
        # Frequency-domain features
        features.extend([
            bandpower(x, fs, 20, 50),
            bandpower(x, fs, 50, 100),
            bandpower(x, fs, 100, 200),
            bandpower(x, fs, 200, 400),
        ])
        
        # Wavelet-energy
        features.extend(wavelet_energy(x))
    
    return np.array(features)

Classifiers¶

Chosen classifiers were

  • Random forest (RF) - as stong baseline classifier that handles feature interactions naturally and is resistant to overfitting
  • Support Vector Machine (SVM) - most popular classifier in modern EMG classification literature. Handles high-dimensional feature vectors well and generalizes robustly with a proper kernel.

from sklearn.metrics import (
    confusion_matrix,
    accuracy_score,
    precision_score,
    recall_score,
    f1_score,
    ConfusionMatrixDisplay,
    classification_report
)
import matplotlib.pyplot as plt

def evaluate_model(model, X_test, y_test, model_name="Model"):
    y_pred = model.predict(X_test)
    
    # Prediction Performance Metrics
    accuracy = accuracy_score(y_test, y_pred)
    precision = precision_score(y_test, y_pred, average='weighted')
    recall = recall_score(y_test, y_pred, average='weighted')
    f1 = f1_score(y_test, y_pred, average='weighted')
    
    print(f"\n {model_name} :")
    print(f"Accuracy:  {accuracy:.4f}")
    print(f"Precision: {precision:.4f}")
    print(f"Recall:    {recall:.4f}")
    print(f"F1-Score:  {f1:.4f}")
    
    print("\nDetailed classification report:")
    print(classification_report(
        y_test, y_pred,
        target_names=['Frown', 'Pucker', 'Smile']
    ))
    
    # Confusion matrix
    cm = confusion_matrix(y_test, y_pred)
    disp = ConfusionMatrixDisplay(
        confusion_matrix=cm,
        display_labels=['Frown', 'Pucker', 'Smile']
    )
    disp.plot(cmap='Blues')
    plt.title(f"{model_name} - Confusion Matrix")
    plt.show()

Results¶

From the results we can see that RF outperformed the SVM notably with the chosen features. RF making only 3 false predictions is a significant difference to SVMs 21 false predictions. SVMs results were not the worst, but with the chosen features and the problem in hand we can see that random forest fitted the problem extremely well.
